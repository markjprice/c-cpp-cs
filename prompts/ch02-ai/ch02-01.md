> **Prompt**: Please give me a checklist to evaluate whether an AI response actually helped me learn C/C++/C# with Unreal or Unity.

Evaluating whether an AI response actually improved your coding skills—rather than just providing a "copy-paste" solution—is crucial for long-term growth in game development.

Use this checklist to audit the quality of the help you receive for **C++, C#, Unity, and Unreal Engine.**

---

## 1. Concept Over Syntax
A good response doesn't just show you *what* to type; it explains *why* the language or engine requires it.

* [ ] **Memory Management:** For C++ or C#, did it explain ownership? (e.g., Why use a `std::unique_ptr` vs a raw pointer, or how Unity’s Garbage Collection handles that specific script?)
* [ ] **Engine Context:** Did it explain the "Life Cycle"? (e.g., Why this code belongs in `InitializeComponent` vs `BeginPlay` in Unreal, or `Awake` vs `Start` in Unity?)
* [ ] **Data Types:** Did it justify the use of specific types? (e.g., Using `float` vs `double` for game physics, or `int32` for Unreal compatibility.)

## 2. Structural & Architectural Logic
Game code can quickly become "spaghetti." A helpful AI should steer you toward scalable patterns.

* [ ] **Design Patterns:** Did the response mention patterns like **Singletons, Observers, or State Machines**?
* [ ] **Decoupling:** Does the solution avoid hard-coding references? (e.g., Using **Interfaces** in C# or **Delegates/Event Dispatchers** in Unreal to let classes talk without being "married" to each other.)
* [ ] **Efficiency:** Did it warn you about expensive operations? (e.g., "Don't put `GetComponent` or `TActorIterator` inside a `Tick`/`Update` function.")

## 3. Tooling and API Literacy
You aren't just learning a language; you're learning a massive library of existing tools.

* [ ] **Native Features:** Did it suggest a built-in engine feature instead of writing custom code? (e.g., Using Unity's **Input System Package** or Unreal's **Enhanced Input**.)
* [ ] **Macros and Attributes:** Did it explain the "magic" keywords? (e.g., `UPROPERTY()` and `UFUNCTION()` in Unreal, or `[SerializeField]` in Unity.)
* [ ] **Debugging:** Did the response suggest how to verify the code? (e.g., Using `DrawDebugSphere` or `Debug.Log`.)

## 4. The "Self-Sufficiency" Test
The ultimate goal of learning is needing the AI less the next time.

* [ ] **Documentation Links:** Did it point you toward specific sections of the **MSDN**, **Unity Manual**, or **Unreal Engine Docs**?
* [ ] **Comparison:** Did it offer an alternative? (e.g., "You could do this in Blueprints/Visual Scripting, but C++/C# is better here because...")
* [ ] **Mental Model:** After reading the response, could you explain the logic to someone else without looking at the code block?

---

### The Red Flag Checklist (If these happen, you aren't learning):
* [ ] The code uses "Magic Numbers" (unexplained constants like `3.14f` or `100`) instead of named variables.
* [ ] It provides a 200-line script without comments explaining the math or logic.
* [ ] It ignores "Best Practices" (e.g., suggesting public variables when private variables with getters/setters are standard).
