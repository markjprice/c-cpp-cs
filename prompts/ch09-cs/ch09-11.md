> **Prompt**: Please show me how pattern matching in modern C# replaces complex `if` statements.

Pattern matching in modern C# (evolving rapidly from C# 7 through C# 11 and 12) has completely transformed how developers handle complex conditional logic.

Historically, evaluating an object against multiple shapes, types, or values forced you into writing deeply nested `if-else` blocks or repetitive, clunky `switch` statements. Pattern matching turns these procedural checks into **declarative expressions**. The compiler inspects the *shape* of your data and maps it to a branch cleanly.

Here is how modern C# pattern matching replaces traditional, complex `if` statements.

---

## 1. Type Testing and Extraction

In older versions of C#, if you wanted to check the type of an object and use it, you had to perform a safe cast (`as`), check if the result was null, and then use the variable.

### The Old Way (`if` with casting)

```csharp
// Clunky, repetitive, and requires declaring variables outside or casting twice
if (vehicle is Car)
{
    Car c = (Car)vehicle;
    c.Drive();
}
else if (vehicle is Bicycle)
{
    Bicycle b = vehicle as Bicycle;
    if (b != null) b.Pedal();
}

```

### The Modern Way (Declaration Pattern)

Using the declaration pattern, C# tests the type and **instantly assigns it to a strongly typed variable** if it matches, all in a single expression.

```csharp
if (vehicle is Car c) c.Drive();
else if (vehicle is Bicycle b) b.Pedal();

```

---

## 2. Replacing Deep Nesting with the Switch Expression

When your logic depends on multiple properties of an object, `if` statements quickly devolve into a "pyramid of doom" (deeply nested brackets).

Imagine calculating a toll price based on a `Vehicle` object's properties:

### The Old Way (Nested `if-else` hell)

```csharp
public decimal GetTollOld(Vehicle v)
{
    if (v is Car)
    {
        var car = (Car)v;
        if (car.Passengers == 0) return 10.00m;
        else if (car.Passengers >= 3) return 5.00m; // Carpool discount
        else return 8.00m;
    }
    else if (v is DeliveryTruck)
    {
        var truck = (DeliveryTruck)v;
        if (truck.GrossWeightInLoads > 5000) return 25.00m;
        else return 15.00m;
    }
    return 2.00m; // Default
}

```

### The Modern Way (Property Patterns & Switch Expressions)

The **switch expression** allows you to strip away all the procedural control flow (`if`, `else`, `return`, brackets) and match directly against the **properties** of the object.

```csharp
public decimal GetTollNew(Vehicle vehicle) => vehicle switch
{
    // Property Pattern: inspects inner values directly
    Car { Passengers: 0 } => 10.00m,
    Car { Passengers: >= 3 } => 5.00m, 
    Car => 8.00m, // Any other car

    // Relational Pattern combines with property pattern
    DeliveryTruck { GrossWeightInLoads: > 5000 } => 25.00m,
    DeliveryTruck => 15.00m,

    null => throw new ArgumentNullException(nameof(vehicle)),
    _ => 2.00m // The discard (_) acts as the 'else' / default block
};

```

## 3. Combining Logic: Relational and Logical Patterns

C# 9 introduced the ability to use logical operators (`and`, `or`, `not`) and relational symbols (`<`, `>`, `<=`, `>=`) inside your patterns. This makes range-checking incredibly elegant compared to standard boolean `if` chains.

### The Old Way (`if` with repeating variables)

```csharp
int age = 25;

if (age >= 0 && age <= 12) 
    Console.WriteLine("Child");
else if (age >= 13 && age <= 19) 
    Console.WriteLine("Teenager");
else if (age == 21 || age == 18)
    Console.WriteLine("Milestone Age");

```

### The Modern Way (Relational Pattern Matching)

You no longer need to repeat the variable name (`age`) over and over again. The compiler infers the subject of the evaluation.

```csharp
string lifeStage = age switch
{
    >= 0 and <= 12 => "Child",
    >= 13 and <= 19 => "Teenager",
    18 or 21 => "Milestone Age",
    _ => "Adult"
};

```

---

## 4. Multi-Variable Matching (Tuple Patterns)

Sometimes an `if` condition relies on the state of multiple independent variables. In the past, this required complex matrixes of `if` and `&&` statements. Tuple patterns allow you to group multiple variables together and match them simultaneously.

Imagine a simple user authentication check or a rock-paper-scissors game logic:

```csharp
// Evaluating two independent variables at once using a tuple
string status = (user.IsLoggedIn, user.Role) switch
{
    (false, _) => "Please log in.",
    (true, "Admin") => "Welcome, Administrator.",
    (true, "Guest") => "Limited access granted.",
    (true, _) => "Welcome back." // Catches any other logged-in role
};

```

## Why Pattern Matching is Better Behind the Scenes

Aside from making your code vastly more readable, pattern matching provides severe hidden advantages:

1. **Compiler Exhaustiveness Checks:** If you use a switch expression on an `enum`, the Roslyn compiler will scan your pattern. If you forget to handle one of the enum values (and didn't provide a default `_` discard), the compiler will throw a **warning**. Standard `if-else` loops cannot do this.
2. **Performance Optimization:** Under the hood, the compiler often optimizes switch expressions into **jump tables** or highly structured IL trees. It can execute significantly faster than evaluating a long chain of independent sequential `if` statements.
