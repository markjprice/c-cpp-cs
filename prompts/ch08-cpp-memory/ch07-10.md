> **Prompt**: Please explain move semantics in simple terms with diagrams.

Think of **Move Semantics** as the difference between **copying a physical document** and **handing someone the original folder.**

In C++, "copying" large objects (like a vector with a million numbers) is expensive because the computer has to allocate new memory and duplicate every single value. Move semantics allows the computer to simply "swap the labels," making the operation nearly instantaneous.

---

### 1. The Analogy: The Warehouse
Imagine you have a warehouse (the **Heap**) full of boxes, and you have a clipboard (an **Object**) with the address of those boxes.

* **Copying:** You build a second warehouse, buy identical boxes, fill them with identical items, and create a second clipboard. This is slow and expensive.
* **Moving:** You just hand your clipboard to someone else and walk away. They now own the warehouse. No boxes were moved; only the "ownership" changed. This is $O(1)$—extremely fast.

---

### 2. How it Works in Memory
When an object is "moved," its internal pointers are copied to the new object, and the original object's pointers are set to `nullptr`.

#### Before the Move:
`Object A` owns a large array at memory address `0x123`.



#### During the Move (`std::move`):
1.  `Object B` copies the address `0x123` from `Object A`.
2.  `Object A` is set to `nullptr` so it doesn't try to delete the memory when it dies.



#### After the Move:
`Object B` now owns the array at `0x123`. `Object A` is empty and "safe" to destroy.



---

### 3. The Code Example
To tell C++ you want to move instead of copy, you use `std::move()`.

```cpp
#include <vector>
#include <iostream>

int main() {
    std::vector<int> listA = {1, 2, 3, 4, 5}; // A owns the data

    // std::move doesn't move data; it CASTS listA to a "movable" type.
    std::vector<int> listB = std::move(listA); 

    // listB now has the 5 integers.
    // listA is now EMPTY.
    std::cout << "List B size: " << listB.size() << "\n";
    std::cout << "List A size: " << listA.size() << "\n";
}
```

---

### 4. When is this used?
Move semantics are triggered automatically in two main cases:
1.  **Temporary Objects:** When you return a large object from a function, C++ "moves" it out instead of copying it.
2.  **Explicit Moves:** When you are done with an object and want to pass its "guts" to another object (like passing a task to a background thread).

### Summary Table

| Feature | Copying | Moving |
| :--- | :--- | :--- |
| **Speed** | Slow ($O(n)$) | Instant ($O(1)$) |
| **Memory** | Uses twice as much RAM | Uses no extra RAM |
| **Original Object** | Remains unchanged | Becomes empty/null |
| **C++ Tool** | Copy Constructor | Move Constructor (`&&`) |
