> **Prompt**: Compare shallow copy vs deep copy in C++ with memory diagrams.

In C++, the difference between a **shallow copy** and a **deep copy** becomes critical when your class manages resources on the **Heap** (like dynamic arrays or objects created with `new`).

If you don't explicitly define how your class should be copied, C++ provides a default "member-wise" copy, which results in a shallow copy.

---

## 1. Shallow Copy (The Default)
A shallow copy copies the **values** of the member variables exactly as they are. If one of those members is a pointer, it copies the **memory address**, not the data at that address.

* **Result:** Two different objects end up pointing to the **same** memory location on the heap.
* **The Danger:** When one object is destroyed, its destructor frees the memory. When the second object is destroyed, it tries to free the same memory, causing a **Double Free Crash**.



### Example Code (Shallow):
```cpp
class Shallow {
public:
    int* data;
    Shallow(int val) { data = new int(val); }
    // No copy constructor defined - Compiler creates a shallow one!
    ~Shallow() { delete data; } 
};

// In main:
Shallow obj1(10);
Shallow obj2 = obj1; // Both now point to the same memory address.
```

---

## 2. Deep Copy (The Safe Way)
A deep copy allocates **new memory** on the heap for the copy and then duplicates the actual data from the source into the new location.

* **Result:** Each object has its own independent copy of the data.
* **The Benefit:** Changes to one object do not affect the other, and they can be destroyed safely without interfering with each other's memory.



### Example Code (Deep):
To achieve a deep copy, you must write your own **Copy Constructor**.

```cpp
class Deep {
public:
    int* data;
    Deep(int val) { data = new int(val); }

    // User-defined Copy Constructor
    Deep(const Deep& other) {
        data = new int(*other.data); // 1. Allocate new memory. 2. Copy value.
    }

    ~Deep() { delete data; }
};

// In main:
Deep obj1(10);
Deep obj2 = obj1; // Separate memory addresses, same values.
```

---

## 3. Comparison Summary

| Feature | Shallow Copy | Deep Copy |
| :--- | :--- | :--- |
| **Pointers** | Copies the address (pointer value). | Copies the actual data pointed to. |
| **Heap Memory** | Shared between objects. | Unique to each object. |
| **Destruction** | Likely to cause a **Double Free**. | Safe and independent. |
| **Efficiency** | Faster (no allocation). | Slower (requires allocation/copying). |
| **Default** | Yes (Default C++ behavior). | No (Must be manually implemented). |

---

## 4. The "Rule of Three"
If your class requires a deep copy, it almost certainly follows the **Rule of Three**. This rule states that if you need to define any one of the following, you likely need to define all three:
1.  **Destructor** (to free the memory).
2.  **Copy Constructor** (to perform the deep copy).
3.  **Copy Assignment Operator** (to handle `obj1 = obj2` scenarios).



### Modern Alternative: The Rule of Zero
In modern C++, you can avoid this headache entirely by using `std::vector` or `std::string` instead of raw pointers. These Standard Library classes handle their own deep copies and memory management automatically!
