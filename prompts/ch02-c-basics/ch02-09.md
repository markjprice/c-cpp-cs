### **Question 1: Post-Increment Confusion**
What is the output of the following code snippet?
```c
int x = 5;
int y = x++;
printf("%d, %d", x, y);
```
**A)** `5, 5`  
**B)** `6, 5`  
**C)** `6, 6`  
**D)** `5, 6`

> **Answer: B** > **Explanation:** The `++` operator is in **postfix** position. This means the current value of `x` (5) is assigned to `y` *first*, and only then is `x` incremented to 6. If it were `++x` (prefix), the answer would be `6, 6`.

### **Question 2: The Logical "Shortcut"**
What will be the value of `count` after this code executes?
```c
int count = 0;
int a = 0;
if (a && ++count) {
    count += 10;
}
printf("%d", count);
```
**A)** `0`  
**B)** `1`  
**C)** `11`  
**D)** Compilation Error

> **Answer: A** > **Explanation:** This demonstrates **short-circuit evaluation**. In a logical AND (`&&`), if the left side is false (`0`), C does not evaluate the right side at all. Because `a` is 0, `++count` is never executed, leaving `count` at 0.

### **Question 3: Precedence vs. Associativity**
What is the result of the following expression?
```c
int result = 10 - 4 - 2;
```
**A)** `8`  
**B)** `4`  
**C)** `0`  
**D)** `12`

> **Answer: B** > **Explanation:** When operators have the same precedence (like subtraction), C uses **associativity**. Subtraction is left-to-right associative, so the calculation is `(10 - 4) - 2`, which equals `4`.

### **Question 4: Integer Division & Promotion**
What does this code print?
```c
float d = 5 / 2;
printf("%.1f", d);
```
**A)** `2.5`  
**B)** `2.0`  
**C)** `3.0`  
**D)** `0.0`

> **Answer: B** > **Explanation:** In C, if both operands are integers, the operation is **integer division**. `5 / 2` results in `2` (the remainder is discarded). Only *after* the division is the result converted to a float. To get `2.5`, you would need to write `5.0 / 2`.

### **Question 5: The Ternary Challenge**
What is the value of `z`?
```c
int x = 10, y = 20;
int z = (x > y) ? x : y + 5;
```
**A)** `10`  
**B)** `20`  
**C)** `25`  
**D)** `15`

> **Answer: C** > **Explanation:** The condition `(x > y)` is false because 10 is not greater than 20. Therefore, the ternary operator selects the second expression: `y + 5`. Since `y` is 20, the result is 25.

### **Summary Table: Quick Operator Rules**

| Feature | Rule to Remember |
| :--- | :--- |
| **`&&` and `||`** | Stop evaluating as soon as the result is known (Short-circuit). |
| **`x++` vs `++x`** | Postfix returns the old value; Prefix returns the new value. |
| **Integer `/`** | Always truncates decimals; does not round. |
| **`%` (Modulo)** | Only works with integer types. |
| **`=` vs `==`** | `=` is for assignment; `==` is for comparison. |
