> **Prompt**: Rewrite a messy C program into clean, readable code and explain the improvements.

To illustrate the power of clean code, let's take a "messy" program that calculates the area of a circle and determines if it's "large" or "small," then transform it into professional-grade C.

### The Messy Version
```c
#include <stdio.h>
#define P 3.14159
int main(){float r;printf("Enter r: ");scanf("%f",&r);float a=P*r*r;
if(a>100){printf("Large: %f\n",a);}else{printf("Small: %f\n",a);}return 0;}
```
**Why this is "Messy":**
* **Cryptic Names:** Variables like `P`, `r`, and `a` force the reader to guess their meaning.
* **Poor Formatting:** No whitespace or indentation makes the logic bleed together.
* **Magic Numbers:** The value `100` is hardcoded without explanation.
* **Lack of Structure:** Everything is crammed into `main`, making it hard to reuse.

---

### The Clean Version
```c
#include <stdio.h>
#include <stdbool.h>

/**
 * Clean C: Circle Area Calculator
 * Demonstrates: Constant usage, descriptive naming, and modularity.
 */

#define PI 3.14159f
#define AREA_THRESHOLD 100.0f

// Function to calculate circle area
float calculate_circle_area(float radius) {
    return PI * (radius * radius);
}

int main() {
    float radius = 0.0f;
    float area = 0.0f;

    printf("Please enter the circle radius: ");
    if (scanf("%f", &radius) != 1) {
        printf("Error: Invalid input.\n");
        return 1;
    }

    area = calculate_circle_area(radius);

    // Determine size category based on the defined threshold
    if (area > AREA_THRESHOLD) {
        printf("The circle is Large (Area: %.2f)\n", area);
    } else {
        printf("The circle is Small (Area: %.2f)\n", area);
    }

    return 0;
}
```

---

### Key Improvements Explained

#### 1. Descriptive Naming
We replaced `r` with `radius` and `a` with `area`. In C, your code should be "self-documenting." A programmer looking at this six months from now won't have to hunt for definitions to understand that `radius` represents a measurement.

#### 2. Encapsulation (Functions)
By moving the math into `calculate_circle_area`, we've isolated the logic. If the formula for an area changes (or we want to use it elsewhere), we only change it in one place.

#### 3. Named Constants
Instead of a "magic number" like `100`, we used `#define AREA_THRESHOLD`. This gives the number **semantic meaning**. If the definition of a "Large" circle changes to $150$, we update one line at the top rather than searching through the logic.

#### 4. Error Handling
The messy version assumes the user always types a number. The clean version checks the return value of `scanf`. This prevents the program from processing "garbage" data if a user types a letter instead of a number.

#### 5. Formatting and Whitespace
Proper indentation and vertical spacing (empty lines between logical blocks) guide the reader's eye. It distinguishes between the setup, the logic, and the output.
