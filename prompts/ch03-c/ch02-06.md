> **Prompt**: Demonstrate short-circuit evaluation in C with examples that include side effects.

Short-circuit evaluation is a behavior in C where the compiler skips the evaluation of the second operand in a logical expression if the result is already determined by the first operand.

This isn't just an optimization; it's a fundamental rule of the language that is often used to prevent crashes (like checking if a pointer is null before dereferencing it).

---

## 1. The Rules of Short-Circuiting

C uses two main operators for this:

* **Logical AND (`&&`):** If the left side is **false (0)**, the whole expression must be false. C stops and doesn't look at the right side.
* **Logical OR (`||`):** If the left side is **true (non-zero)**, the whole expression must be true. C stops and doesn't look at the right side.



---

## 2. Examples with Side Effects
A "side effect" occurs when an expression does more than just return a value—like incrementing a variable or printing to the screen. Short-circuiting can cause these side effects to be skipped entirely.

### Example A: The Skipped Increment (AND)
```c
#include <stdio.h>

int main() {
    int x = 0;
    int y = 10;

    // x is 0 (false), so C stops immediately.
    // ++y is NEVER executed.
    if (x && ++y) {
        printf("This won't print.\n");
    }

    printf("x: %d, y: %d\n", x, y); 
    // Output: x: 0, y: 10 (y stayed 10!)
    
    return 0;
}
```

### Example B: The Skipped Function Call (OR)
```c
#include <stdio.h>

int say_hello() {
    printf("Hello from function!\n");
    return 1;
}

int main() {
    int x = 1;

    // x is 1 (true), so the OR condition is satisfied.
    // say_hello() is NEVER called.
    if (x || say_hello()) {
        printf("Condition met.\n");
    }

    return 0;
}
// Output: Condition met. (Notice "Hello from function!" is missing)
```

## 3. Real-World Safety Pattern
The most common "pro" use of short-circuiting is the **Guard Clause**. It allows you to check for a valid state and perform an action in a single line without crashing.

```c
#include <stdio.h>

int main() {
    int *ptr = NULL;

    // Without short-circuiting, ptr[0] would crash (segfault).
    // Because ptr == NULL is true, C stops before touching ptr[0].
    if (ptr != NULL && ptr[0] == 5) {
        printf("Found the value.\n");
    } else {
        printf("Pointer is null or value is not 5.\n");
    }

    return 0;
}
```


---

## Summary of Behavior

| Operator | Left Side Value | Right Side Evaluated? | Why? |
| :--- | :--- | :--- | :--- |
| `&&` | `0` (False) | **No** | Result is guaranteed to be `0`. |
| `&&` | `1` (True) | **Yes** | Result depends on the right side. |
| `||` | `1` (True) | **No** | Result is guaranteed to be `1`. |
| `||` | `0` (False) | **Yes** | Result depends on the right side. |

**Word of Caution:** Avoid putting critical logic (like `x = perform_math()`) on the right side of a logical operator unless you *want* it to be skipped occasionally. It's a common source of "ghost bugs" where variables don't update when you expect them to.
