> **Prompt**: Please explain how a C program is executed step-by-step from source code to running executable, including preprocessing and compilation stages.

The transformation of a C program from a simple text file into a running process involves four distinct stages, collectively known as the **Compilation Pipeline**.

---

## 1. Preprocessing
The preprocessor (`cpp`) handles lines starting with a `#`. It doesn't understand C syntax; it's essentially a sophisticated text-manipulation tool.

* **Macro Expansion:** Replaces macro names (e.g., `#define PI 3.14`) with their values.
* **File Inclusion:** Replaces `#include <stdio.h>` with the actual contents of that header file.
* **Conditional Compilation:** Processes blocks like `#ifdef` or `#if`.
* **Result:** A **translation unit** (usually a `.i` file), which is pure C code with all headers expanded and comments removed.



---

## 2. Compilation
The compiler (e.g., the `cc1` component of GCC) takes the preprocessed code and translates it into a lower-level language that the CPU can eventually understand.

* **Analysis:** The compiler checks for syntax errors and ensures the code follows C language rules.
* **Optimization:** It rearranges code to make it run faster or use less memory.
* **Result:** An **assembly file** (usually a `.s` file). Assembly code is human-readable but specific to the computer's processor architecture (like x86_64 or ARM).

---

## 3. Assembly
The assembler (`as`) converts the assembly code into machine code (binary).

* **Instruction Mapping:** It translates mnemonic instructions (like `MOV` or `ADD`) into their equivalent binary opcodes.
* **Object File:** The output is an **object file** (usually a `.o` or `.obj` file). This file contains machine code, but it is "relocatable"—it doesn't know where it will sit in memory yet, and it doesn't know the addresses of functions located in other files (like `printf`).



---

## 4. Linking
The linker (`ld`) is the final architect. It merges all the individual object files and any required libraries into a single cohesive file.

* **Symbol Resolution:** If your code calls `printf`, the linker finds the machine code for `printf` in the standard C library and "links" it to your program.
* **Address Assignment:** It calculates the final memory addresses for all functions and variables.
* **Result:** The **Executable file** (e.g., `a.out`, `main.exe`).

---

## 5. Execution (Loading)
When you run the program, the Operating System's **Loader** takes over:

1.  **Memory Allocation:** The OS allocates space in RAM for the program's code, global variables, stack, and heap.
2.  **Copying:** The loader copies the machine code from the disk into the allocated RAM.
3.  **Jump to Entry Point:** The CPU is instructed to start executing at a specific address (usually a startup routine that eventually calls your `main()` function).
