In C, every integer type has a specific "capacity" based on how many bits it uses. **Integer overflow** occurs when you try to store a value that is larger than the maximum (or smaller than the minimum) that the variable type can hold.

Think of it like a car's **odometer**: when it hits the maximum value (e.g., 999,999), adding one more mile causes it to "wrap around" back to 000,000.

---

## 1. How It Works (The "Wrapping" Effect)
Most modern systems use a system called **Two's Complement** to represent integers. When a value exceeds the bit limit, the extra bit is simply dropped, leading to unexpected results.

### Signed Integer Overflow
For `signed int` (numbers that can be positive or negative), overflow is technically **undefined behavior** in C. This means the compiler isn't required to do anything specific—it might wrap around, it might crash, or it might optimize the code in a way that ignores the error. 

Typically, however, adding 1 to the maximum positive value results in the **minimum negative value**.



### Unsigned Integer Overflow
For `unsigned int`, C guarantees "modulo" behavior. It will always wrap back to zero.

---

## 2. Simple Code Examples

### Example: Unsigned Wrap-around
Using `unsigned char` (which typically holds values from 0 to 255):

```c
#include <stdio.h>

int main() {
    unsigned char x = 255;
    printf("Before: %u\n", x); // Prints 255
    
    x = x + 1; 
    printf("After:  %u\n", x); // Prints 0 (Wrapped around!)
    
    return 0;
}
```

### Example: Signed Overflow
Using a standard `signed int` and the `limits.h` library to find the maximum value:

```c
#include <stdio.h>
#include <limits.h>

int main() {
    int max = INT_MAX; // Usually 2,147,483,647
    printf("Max: %d\n", max);
    
    int overflowed = max + 1;
    printf("Max + 1: %d\n", overflowed); // Usually prints -2,147,483,648
    
    return 0;
}
```

---

## 3. Why It Is Dangerous
Integer overflow is a common source of bugs and security vulnerabilities. 

* **Logic Errors:** A program checking if `(price1 + price2) > budget` might fail if the sum overflows and becomes a small or negative number.
* **Infinite Loops:** If a loop counter overflows before it reaches the "stop" condition, the loop may never end.
