> **Prompt**: Please show me how different compilers like GCC, Clang, and MSVC report the same syntax error in a simple C program.

To see how different compilers handle errors, let's use a classic mistake: a **missing semicolon** after a variable declaration. This is a syntax error because it violates the formal grammar of the C language.

### The Sample Code (`main.c`)

```c
#include <stdio.h>

int main() {
    int x = 10
    printf("%d\n", x);
    return 0;
}
```

---

### 1. GCC (GNU Compiler Collection)
GCC is known for being direct and traditional. It focuses on the specific token where the parser got confused.

**The Output:**
```text
main.c: In function 'main':
main.c:5:5: error: expected ';' before 'printf'
    5 |     printf("%d\n", x);
      |     ^~~~~~
```
* **Style:** It identifies the line and column ($5:5$) and tells you exactly what it was looking for (`expected ';'`) before the next command.

---

### 2. Clang (LLVM)
Clang was designed specifically to provide "expressive" diagnostics. It often uses colors and "caret" diagnostics to be more user-friendly.

**The Output:**
```text
main.c:4:15: error: expected ';' at end of declaration
    int x = 10
              ^
              ;
```
* **Style:** Instead of pointing to the next line (like GCC often does), Clang points to the **exact spot** where the semicolon is missing. It even provides a visual suggestion of where to insert the character.

---

### 3. MSVC (Microsoft Visual C++)
MSVC is the standard for Windows development. Its error messages are often structured to integrate well with the Visual Studio IDE's "Error List" window.

**The Output:**
```text
main.c(5): error C2146: syntax error: missing ';' before identifier 'printf'
```
* **Style:** It uses a specific error code (**C2146**), which is very helpful for looking up detailed documentation on Microsoft Learn. It is less "visual" than Clang but very precise about the identifier that triggered the failure.

---

### Comparison Summary

| Feature | GCC | Clang | MSVC |
| :--- | :--- | :--- | :--- |
| **Precision** | Points to the next line/token. | Points to the specific gap. | Points to the next token. |
| **Visuals** | Basic ASCII underlines. | Highly descriptive "fix-it" hints. | Text-heavy, code-focused. |
| **Searchability** | General text. | General text. | **Error Codes** (e.g., C2146). |
