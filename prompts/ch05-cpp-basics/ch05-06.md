> **Prompt**: Please compare pass-by-value vs pass-by-reference vs pass-by-const-reference with performance examples.

In C++, how you pass arguments to a function dictates whether you are making a copy of your data or just pointing to the original. This is the primary lever for performance optimization in modern C++.

---

## 1. Pass-by-Value
When you pass by value, the compiler creates a **complete copy** of the object. For a simple `int`, this is fast. For a `std::vector` with a million elements, this is a performance disaster.

* **Behavior:** The function gets its own private copy. Changes made inside the function do not affect the original.
* **Cost:** Memory allocation + copying all data.

```cpp
void process(std::string s) { // A copy is made here
    s += " updated"; 
}
```

---

## 2. Pass-by-Reference
Instead of copying the data, you pass an **alias** (a reference) to the existing object. 

* **Behavior:** The function works directly on the original object. Changes made inside the function **will** affect the original.
* **Cost:** Near zero (it's essentially passing a pointer under the hood).



```cpp
void process(std::string& s) { // No copy; works on original
    s += " updated"; 
}
```

---

## 3. Pass-by-Const-Reference
This is the "Golden Rule" for passing large objects in C++. You get the performance of a reference (no copying) but the safety of pass-by-value (the function cannot modify the original).

* **Behavior:** Read-only access to the original object.
* **Cost:** Near zero.

```cpp
void process(const std::string& s) { 
    // s += " error"; // Won't compile! It's read-only.
    std::cout << s; 
}
```

---

## Performance Comparison

Imagine passing a `std::vector<int>` containing **1 million integers** (roughly 4MB of data).

| Method | Time Complexity | Operation | Use Case |
| :--- | :--- | :--- | :--- |
| **Pass-by-Value** | $O(N)$ | Allocates 4MB, copies 1M ints | Small types (int, double, char) |
| **Pass-by-Ref** | $O(1)$ | Passes an 8-byte address | When you *need* to modify the original |
| **Pass-by-Const-Ref** | $O(1)$ | Passes an 8-byte address | **Large objects** (strings, vectors, classes) |

### Real-World Example: A Search Function
If you are searching for a value in a large dataset, using pass-by-value would mean every time you call `search()`, your program pauses to copy the entire database.

```cpp
// ❌ Disaster: Copies the whole vector every call
bool findInVector(std::vector<int> vec, int target) { ... }

// ✅ Efficient: No copy, but safe from modification
bool findInVector(const std::vector<int>& vec, int target) { ... }
```

---

## The "When to Use What" Rule of Thumb

1.  **Use Value** for small, "Trivial" types (anything $\le$ 16 bytes, like `int`, `float`, `bool`, or `std::string_view`).
2.  **Use Const-Reference** for everything else (Objects, Strings, Vectors, Maps).
3.  **Use Reference** only when the explicit purpose of the function is to modify the input variable (e.g., a `swap` function).
