> **Prompt**: Please explain ownership of memory in C with a diagram and examples.

In C, **ownership** is a conceptual contract rather than a language feature. Unlike languages like Rust, where the compiler tracks who "owns" a piece of memory, C leaves this entirely to the programmer. 

If you allocate memory on the heap, you are the owner. If you pass that pointer to another function, you must decide: are you **lending** it or **transfering** it?

---

## 1. The Ownership Lifecycle
Ownership is essentially the responsibility to `free()` a pointer exactly once. The lifecycle typically follows these stages:

1.  **Acquisition:** Memory is allocated via `malloc()` or `calloc()`.
2.  **Usage:** The pointer is used to read or write data.
3.  **Transfer (Optional):** Responsibility for the memory is passed to another module.
4.  **Release:** The owner calls `free()`.



---

## 2. Lending vs. Transferring
The primary source of bugs in large C programs is confusion between "Lending" and "Transferring."

### Lending (Borrowing)
When you lend a pointer, the caller retains ownership. The function being called is promised that the memory will exist for the duration of the call, but it **must not** free it or store it for later use.

```c
void print_name(const char *name) {
    // I am "borrowing" name.
    printf("Name: %s\n", name);
    // I do NOT call free(name) here.
}
```

### Transferring (Hand-off)
When you transfer ownership, the original owner "forgets" about the pointer. The receiver is now solely responsible for eventually freeing it.

```c
char* create_greeting(const char *user) {
    char *msg = malloc(50);
    sprintf(msg, "Hello, %s", user);
    return msg; // Ownership is transferred to the caller.
}

int main() {
    char *my_msg = create_greeting("Alice");
    // ... use my_msg ...
    free(my_msg); // main() is the new owner and must free it.
}
```

## 4. Professional Workarounds
To manage ownership in massive codebases, C developers use these conventions:

* **Prefixes:** A function like `Buffer_create()` implies the caller becomes the owner. `Buffer_use()` implies lending.
* **The "Alloc/Free" Pair:** If a module has an `Init()` function that allocates memory, it **must** provide a `Deinit()` function. Users of the module should never call `free()` on the module's internals directly.
* **Documentation:** Comments like `/* @transfer ownership */` or `/* @borrow */` are used because the compiler cannot help you.
